## 1. LLM（大語言模型）

**大白話解釋：**
它本質上是一個超級巨大的「接龍文字遊戲機」。透過閱讀海量的網路文字，它學會了預測「給定上文之後，最可能出現哪個詞」。

**生動例子：**
輸入「床前明月光」，LLM 就會根據概率猜測下一個詞是「疑是地上霜」。

> **⚠️ 常見誤區**：很多人以為 LLM 像人類一樣有自主意識或真正的「邏輯思考能力」。實際上，它只是在做高級的數學概率計算，並沒有真正理解文字的語意。

---

## 2. Context Window（上下文窗口）

**大白話解釋：**
這是 AI 的「短期記憶力」。它決定了你在一輪對話中，最多能塞給 AI 多少字——包括你傳入的內容和它回覆的內容，全部加總計算。

**生動例子：**
如果一個模型的上下文窗口是 8k token（約 6,000 字），你傳了一篇 1 萬字的小說請它總結，它就會「丟三落四」，因為超過了它的短期記憶上限。

> **⚠️ 使用邊界**：雖然現在很多模型宣稱有 100k 甚至 1M 的超長上下文，但仍存在「**迷失在中央**（Lost in the Middle）」的現象——AI 容易記住開頭和結尾，但容易遺漏中間部分的細節，使用時需特別注意重要資訊的擺放位置。

---

## 3. Prompt（提示詞）

**大白話解釋：**
你對 AI 發出的指令、設定或問題。它是你與 AI 溝通的媒介，直接決定了 AI 輸出的品質——垃圾輸入，垃圾輸出。

**生動例子：**

| 品質 | Prompt 範例 |
|------|-------------|
| ❌ 差 | 寫一封信 |
| ✅ 好 | 你現在是一位擁有 10 年經驗的 HR，請給未通過面試的候選人寫一封語氣委婉的拒信，字數 150 字以內 |

> **⚠️ 常見誤區**：以為 Prompt 是萬能魔法。如果模型本身沒有這項知識（例如問它一個從未出現在訓練資料中的事實），再完美的 Prompt 也無法憑空變出正確答案——它只會「一本正經地胡說八道」（Hallucination）。

## 4. Tool Use（工具調用 / Function Calling）

**大白話解釋：**
AI 雖然會聊天，但它不會算複雜的帳、不認路、也不知道今天的即時天氣。工具調用就是給 AI 插上機械臂，讓它在需要時去呼叫外部系統（如計算器、天氣 API、資料庫）。

**生動例子：**
你問 AI「今天北京氣溫加上海氣溫一共幾度？」，AI 自己不會算，但它會舉手說：「我需要呼叫【天氣 API】查溫度，再呼叫【計算器】求和」，然後把參數整理好。

> **⚠️ 使用邊界**：AI 本身**不執行**工具，它只負責「決定要用什麼工具」並把參數整理好，真正的執行是由背後的程式碼完成的。如果網路超時或 API 掛掉，AI 也會束手無策，這部分的錯誤處理必須由開發者自行設計。

---

## 5. Workflow（工作流）

**大白話解釋：**
把一個複雜的任務，拆解成固定好的、按部就班的流水線步驟。在這條流水線上，AI 只負責其中某些特定環節的加工，其餘步驟由程式碼控制。

**生動例子：**
一個「AI 自動寫週報」的工作流：

```
Step 1  從 Git 抓取本週程式碼提交記錄      ← 程式碼執行
Step 2  把記錄交給 AI 總結成摘要            ← AI 負責
Step 3  把摘要自動發送到釘釘群              ← 程式碼執行
```

> **⚠️ 常見誤區**：認為工作流就是讓 AI 自由發揮。恰恰相反，優秀的工作流是為了**限制** AI 的隨機性，用確定性的規則（Deterministic Rules）來規範 AI 的輸出，讓整個流程可預期、可重現。

---

## 6. Agent（智能體）

**大白話解釋：**
工作流走固定流程，而 Agent 則是把「決定權」交給 AI。它擁有目標、記憶、規劃能力和工具，能自己決定「第一步做什麼，如果失敗了第二步怎麼調整」，持續循環直到達成目標。

**生動例子：**

| 方式 | 遇到機票售完時的反應 |
|------|----------------------|
| **Workflow** | 流程中斷，報錯退出 |
| **Agent** | 自己思考：「那我查明天的票？或換個航空公司？」持續嘗試直到達成目標 |

> **⚠️ 使用邊界**：Agent 因為具備「自主循環思考」的特性，非常容易陷入死循環（Infinite Loop），或因嘗試次數過多導致 Token 費用爆炸。目前在複雜商業場景中，其穩定性仍需謹慎評估，通常需要設定最大循環次數（Max Iterations）作為安全閘門。

## 概念關係總覽

```
你的輸入 (Prompt)
    │
    ▼
┌─────────────────────────────────────┐
│           LLM（大腦）                │
│   受限於 Context Window（記憶大小）  │
└────────────┬────────────────────────┘
             │ 需要外部能力時
             ▼
         Tool Use（工具調用）
         ├── 計算器
         ├── 天氣 API
         └── 資料庫 ...

組合起來：
  固定步驟 → Workflow（工作流）
  自主決策 → Agent（智能體）
```
